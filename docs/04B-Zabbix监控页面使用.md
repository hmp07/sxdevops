# SxDevOps Zabbix 监控页面使用

## 一、概述

Zabbix 监控页面提供对 Zabbix Server 的实时只读视图，可以在平台内查看主机状态、监控项、触发器和当前问题，无需登录 Zabbix 前端。

**前置条件：**
- 已配置至少一个 [Zabbix 数据源](04A-Zabbix数据源配置与管理.md)
- 需要 `ops.zabbix.view` 权限

## 二、页面入口

侧边栏：**可观测性 → Zabbix 监控**

## 三、页面布局

### 3.1 顶部区域

- **数据源选择器**：下拉框列出所有已配置的 Zabbix 数据源，自动选中默认数据源
- **刷新按钮**：手动刷新所有数据

### 3.2 统计卡片

| 统计项 | 说明 |
|--------|------|
| 主机总数 | 当前数据源监控的主机数量 |
| 可用主机 | 主接口 `available != 2` 的主机数 |
| 当前问题 | 活跃的告警问题数量 |

### 3.3 三个 Tab

| Tab | 内容 |
|-----|------|
| **主机列表** | 所有主机及其状态、IP、主机组、关联 CI |
| **触发器** | 触发器列表及其严重度与状态 |
| **当前问题** | Zabbix 中活跃的告警问题 |

## 四、主机列表

### 4.1 表格列

| 列 | 说明 |
|----|------|
| 主机名 | Zabbix 中的技术名称 |
| 可见名称 | Zabbix 中的显示名称 |
| 状态 | 可用 / 不可用（通过主接口 available 字段判断） |
| IP | 主接口 IP 地址 |
| 主机组 | 所属主机组标签（最多显示 3 个） |
| 关联 CI | CMDB 中关联的配置项名称（未关联显示"未关联"） |

### 4.2 操作

- **搜索主机**：通过搜索框按主机名或可见名称过滤
- **点击主机行**：弹出监控项详情对话框
- **分页**：支持 10/20/50/100 条每页

### 4.3 监控项详情对话框

点击主机行后弹出，显示该主机所有监控项：

| 列 | 说明 |
|----|------|
| 监控项名称 | 如 CPU utilization、Free memory |
| 键值 | Zabbix item key |
| 最新值 | 最近一次采集的数值 |
| 单位 | 如 %、B、bps |
| 状态 | 启用 / 禁用 |

## 五、触发器 Tab

| 列 | 说明 |
|----|------|
| 触发器名称 | 告警规则描述 |
| 严重度 | 颜色标签：未分类 / 信息 / 警告 / 一般严重 / 严重 / 灾难 |
| 状态 | 触发（红色）/ 正常（绿色） |
| 关联主机 | 触发器关联的主机 |

## 六、当前问题 Tab

| 列 | 说明 |
|----|------|
| 问题描述 | 告警名称 |
| 严重度 | 颜色标签 |
| 确认 | 已确认 / 未确认 |
| 时间 | 问题发生时间 |

## 七、设备关联映射

系统在加载主机列表时自动执行设备匹配——通过 IP 地址将 Zabbix 主机与 CMDB 配置项关联。

匹配逻辑：
1. 提取 Zabbix 主机主接口 IP
2. 在 CMDB 配置项的 `attributes` 中搜索匹配的 `ip_address` 或 `managementip`
3. 匹配成功 → 创建 `DeviceMapping` 记录，在主机列表"关联 CI"列显示配置项名称
4. 已有映射的主机直接显示关联结果

## 八、使用场景

| 场景 | 操作方式 |
|------|---------|
| 巡检 | 查看主机列表 → 确认所有主机状态正常 |
| 排障 | 查看当前问题 → 找到告警主机 → 点击查看监控项详情 |
| 容量分析 | 查看特定主机监控项 → 关注 CPU/内存/磁盘使用 |
| CMDB 关联 | 确认"关联 CI"列 → 未关联的可通过 CMDB 手动关联 |
