# SxDevOps 本地开发环境搭建

## 一、概述

本地开发模式前后端分离运行：Django 后端使用 SQLite（无需 MySQL/Redis），Vue 前端通过 Vite 代理 API 请求。适合代码开发和调试。

**前置条件：**
- Python 3.10+（推荐 3.12）
- Node.js 20+（推荐 22 LTS）
- Git Bash 或 WSL（Windows 下推荐）

## 二、后端启动

```bash
cd backend

# 1. 安装 Python 依赖
pip install -r requirements.txt

# 2. 执行数据库迁移（自动创建 backend/db.sqlite3）
python manage.py migrate

# 3. 创建演示数据（可选）
python manage.py seed_data

# 4. 创建 AI 智能体模板（可选）
python manage.py seed_templates

# 5. 启动 Daphne ASGI 服务器
python -m daphne -b 0.0.0.0 -p 8000 sxdevops.asgi:application
```

后端启动后访问 `http://127.0.0.1:8000`，API 根路径返回 401（需要认证，正常现象）。

## 三、前端启动

```bash
cd frontend

# 1. 安装 Node 依赖
npm install

# 2. 启动 Vite 开发服务器
npm run dev
```

前端启动后访问 `http://127.0.0.1:3000`。

## 四、开发模式架构

```
浏览器 :3000  ──→ Vite Dev Server  ──→ Vue 3 (HMR 热更新)
                    │
                    │ /api/* 代理
                    ▼
浏览器 :3000  ──→ Daphne :8000  ──→ Django REST API
                                     │
                                     ├── SQLite (backend/db.sqlite3)
                                     └── 内存缓存 + 内存 Channel Layer
```

## 五、数据库配置

本地开发默认使用 SQLite，数据库文件位于 `backend/db.sqlite3`，无需额外配置。

如需本地使用 MySQL 或 Redis，创建 `backend/config.json`（参考 `backend/config.example.json`）：

```json
{
  "database": {
    "type": "mysql",
    "host": "127.0.0.1",
    "port": 3306,
    "database_name": "sxdevops",
    "user": "root",
    "password": "your-password",
    "charset": "utf8mb4",
    "conn_max_age": 60
  },
  "cache": {
    "type": "redis",
    "redis_url": "redis://127.0.0.1:6379/0"
  }
}
```

> 注意：`config.json` 已在 `.gitignore` 中排除，不会提交到版本库。

## 六、Vite 代理配置

`frontend/vite.config.js` 中的代理设置：

| 路径 | 代理目标 |
|------|---------|
| `/api` | `http://127.0.0.1:8000` |
| `/ws` | `ws://127.0.0.1:8000` (WebSocket) |

## 七、常用命令速查

```bash
# 后端
cd backend
python manage.py test                 # 运行测试
python manage.py test ops             # 运行单个 app 测试
python manage.py makemigrations       # 生成迁移文件
python manage.py migrate              # 应用迁移
python manage.py seed_data            # 重新生成演示数据

# 前端
cd frontend
npm run build                         # 生产构建
npm run preview                       # 预览生产构建

# Windows 快速启动
.\tools\dev\start-dev.ps1             # 启动前后端
.\tools\dev\stop-dev.ps1              # 停止前后端
```

## 八、开发调试提示

- 后端代码修改后，Daphne 不会自动重载。建议开发时使用 `python manage.py runserver` 替代 Daphne（但不支持 WebSocket）
- 前端代码修改后，Vite HMR 自动生效，浏览器即时刷新
- 默认账号与 Docker 部署相同：`admin / Admin@123456`
- 本地 SQLite 并发写入有限制，如遇到 `database is locked` 错误，重启 Daphne 即可
