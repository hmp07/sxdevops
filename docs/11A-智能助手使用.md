# SxDevOps 智能助手使用

## 一、概述

AIOps 智能助手是平台的 AI 交互入口，支持自然语言排障、工具调用、知识图谱查询和运维动作生成。

**入口：**
- 侧边栏：**AIOps → 智能助手**（全屏模式）
- 右下角浮动按钮（任意页面可用）

**权限：** `aiops.chat.view`

## 二、使用方式

### 2.1 发起对话

在输入框中用自然语言描述问题：

```
"帮我查下生产环境 Zabbix 上有没有严重告警"
"分析 order-center 服务最近的错误日志"
"这个变更会影响哪些服务器"
"生成数据库巡检任务"
```

### 2.2 理解响应

AI 采用二阶段应答模式：
1. **工具调用阶段**：AI 通过 MCP 工具查询平台数据（告警、日志、指标等）
2. **答案生成阶段**：AI 基于查询结果生成结构化答案

响应包含：
- **结论**：核心发现
- **证据**：支撑结论的数据来源
- **风险提示**：潜在风险
- **建议动作**：推荐的后续操作

### 2.3 待确认动作

如果 AI 建议执行写入操作（如生成任务、创建工单），需要用户确认后才执行。确认项显示在对话中，点击确认后由平台 API 执行。

## 三、支持的排障场景

| 场景 | 触发方式 |
|------|---------|
| Zabbix 告警分析 | "分析 Zabbix 告警" → React 代理 + Zabbix 工具 |
| 跨系统根因分析 | "分析这个故障的根因" → Zabbix + 告警 + 日志 + 链路 + 变更 |
| iTop 变更影响分析 | "这个变更影响哪些系统" → iTop CI 关系查询 |

## 四、使用限制

- AI 不能绕过权限和平台 API 直接操作资源
- 写入类操作必须人工确认
- 所有工具调用记录在审计日志中
- 模型调用成本按 Token 统计
