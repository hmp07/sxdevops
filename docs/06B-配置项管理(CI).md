# SxDevOps 配置项管理 (CI)

## 一、概述

配置项（Configuration Item, CI）是 CMDB 的核心数据单元，代表 IT 基础设施中的每个可管理组件——服务器、虚拟机、网络设备、应用、数据库等。

**入口：** 侧边栏 → CMDB → 配置项
**权限：** `cmdb.ci.view` / `cmdb.ci.manage`

## 二、CI 类型

系统预置 CI 类型（可在 CMDB 仪表盘查看和扩展）：

| 类型 | 说明 |
|------|------|
| 服务器 | 物理服务器 |
| 云主机 (ECS) | 云虚拟机 |
| 网络设备 | 交换机、路由器 |
| 存储系统 | SAN、NAS |
| 应用 | 业务应用 |
| 数据库 | 数据库实例 |

## 三、配置项列表

### 3.1 表格列

| 列 | 说明 |
|----|------|
| 名称 | CI 名称 |
| CI 类型 | 类型标签 |
| IP 地址 | 管理 IP |
| 环境 | 生产/测试/开发 |
| 业务线 | 所属业务系统 |
| Zabbix 监控 | 关联的 Zabbix 主机名（未监控显示"未监控"） |
| 状态 | 运行/维护/下线 |

### 3.2 筛选与搜索

- 按 CI 类型、环境、业务线筛选
- 关键字搜索名称和 IP

### 3.3 操作

- **新增配置项**：手动添加（`cmdb.ci.manage` 权限）
- **编辑**：修改 CI 属性
- **删除**：移除 CI 及解除关联关系

## 四、Zabbix 监控关联

配置项列表的 **"Zabbix 监控"** 列显示该 CI 是否已被 Zabbix 监控：
- 关联了 Zabbix 主机 → 显示主机名标签
- 未关联 → 显示 "未监控"

关联通过 `DeviceMapping` 表自动匹配（IP 精确匹配）。可在 Zabbix 监控页面的"关联 CI"列看到反向关联。

## 五、关联关系

CI 之间可以建立关联关系（`CIRelation`），如：
- 服务器 → 运行 → 应用
- 应用 → 依赖 → 数据库
- 网络设备 → 连接 → 服务器

这些关系在资源拓扑页面中可视化展示。
