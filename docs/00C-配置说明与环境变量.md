# SxDevOps 配置说明与环境变量

## 一、配置体系

SxDevOps 支持三层配置，优先级从高到低：

```
环境变量 > config.json > settings.py 默认值
```

配置文件路径可通过 `SXDEVOPS_CONFIG_FILE` 环境变量指定，默认使用 `backend/config.json`。

## 二、数据库配置

### 2.1 引擎选择

| 值 | 数据库 | 适用场景 |
|----|--------|---------|
| `sqlite` | SQLite | 本地开发（默认） |
| `mysql` | MySQL 8.0 | 生产环境（Docker Compose 默认） |

通过 `DATABASE_ENGINE` 环境变量或 `config.json` 的 `database.type` 设置。

### 2.2 MySQL 配置

| 环境变量 | config.json 路径 | 默认值 | 说明 |
|---------|-----------------|--------|------|
| `MYSQL_HOST` | `database.host` | `127.0.0.1` | 主机地址 |
| `MYSQL_PORT` | `database.port` | `3306` | 端口 |
| `MYSQL_DATABASE` | `database.database_name` | — | 数据库名（必填） |
| `MYSQL_USER` | `database.user` | `root` | 用户名 |
| `MYSQL_PASSWORD` | `database.password` | — | 密码 |
| `MYSQL_CHARSET` | `database.charset` | `utf8mb4` | 字符集 |
| `DATABASE_CONN_MAX_AGE` | `database.conn_max_age` | — | 连接池存活时间(秒) |

### 2.3 扩展配置（config.json options）

```json
{
  "database": {
    "type": "mysql",
    "...": "...",
    "options": {
      "init_command": "SET sql_mode='STRICT_TRANS_TABLES'"
    }
  }
}
```

## 三、缓存与 WebSocket 配置

| 环境变量 | 说明 | 默认值 |
|---------|------|--------|
| `CACHE_BACKEND` | 缓存后端：`redis` 或 `local` | 自动判定（有 `REDIS_URL` 则 redis） |
| `REDIS_URL` | Redis 连接 URL | — |
| `CACHE_KEY_PREFIX` | 缓存键前缀 | `sxdevops` |
| `CACHE_DEFAULT_TIMEOUT` | 缓存默认超时(秒) | `300` |
| `CHANNEL_REDIS_URL` | WebSocket 通道层 Redis | 回退到 `REDIS_URL` |
| `CHANNEL_LAYER_BACKEND` | 通道层后端：`redis` 或 `memory` | 自动判定 |

## 四、安全配置

| 环境变量 | 说明 | 默认值 | 生产建议 |
|---------|------|--------|---------|
| `SECRET_KEY` | Django 密钥 | `change-me` | 随机长字符串 |
| `DEBUG` | 调试模式 | `1` | **必须设为 `0`** |
| `ALLOWED_HOSTS` | 允许的主机（逗号分隔） | `*` | 设置为实际域名 |
| `CORS_ALLOW_ALL_ORIGINS` | 允许所有跨域 | `1` | **必须设为 `0`** |
| `SXDEVOPS_PUBLIC_BASE_URL` | 平台公开访问地址 | — | 用于告警卡片回调 URL |

## 五、可观测性集成配置

### 5.1 Grafana

在平台 UI 中配置：可观测性 → 仪表盘 → 设置。

### 5.2 日志数据源

在平台 UI 中配置：日志中心 → 数据源 → 新增数据源。支持：
- **Loki**：填写 Endpoint URL
- **ELK / Elasticsearch**：填写 ES 地址 + 认证方式
- **阿里云 SLS**：填写 Endpoint、Project、Logstore + AccessKey

### 5.3 链路追踪

在平台 UI 中配置：链路追踪 → 数据源 Tab → 新增。支持 SkyWalking、Tempo、Jaeger、Zipkin。

## 六、外部集成配置（Zabbix / iTop）

### 6.1 Zabbix

平台 UI 配置：可观测性 → 数据源 → Zabbix 数据源 Tab → 新增。

| 字段 | 说明 |
|------|------|
| API 地址 | Zabbix JSON-RPC 端点，如 `https://zabbix.example.com/api_jsonrpc.php` |
| 认证方式 | API Token（推荐）或用户名/密码 |
| TLS 验证 | 是否验证 HTTPS 证书 |

> 详细步骤见 [04A-Zabbix数据源配置与管理](04A-Zabbix数据源配置与管理.md)

### 6.2 iTop CMDB

平台 UI 配置：CMDB → iTop 对接 → 新增数据源。

| 字段 | 说明 |
|------|------|
| API 地址 | iTop REST API 端点 |
| 认证方式 | 用户名/密码 |

> 详细步骤见 [05A-iTop数据源配置](05A-iTop数据源配置.md)

## 七、AIOps / LLM 配置

|--------|---------|------|
