# SxDevOps 事件墙查看与过滤

## 一、概述

事件墙是运维操作的统一留痕和复盘中心。所有关键操作——告警触发、任务执行、发布变更、AIOps 工具调用——都会沉淀为事件。

**入口：** 侧边栏 → 事件中心 → 事件中心
**权限：** `eventwall.view`

## 二、事件模型

每个事件记录包含：

| 字段 | 说明 |
|------|------|
| 模块 | 来源模块（ops / aiops / cmdb / eventwall） |
| 类别 | 事件类别（alert / deployment / task / audit） |
| 动作 | 具体操作描述 |
| 结果 | 成功 / 失败 / 部分成功 / 待定 |
| 严重度 | 信息 / 警告 / 危险 |
| 标题 / 摘要 | 事件描述 |
| 资源 | 关联的资源信息 |
| 时间 | 事件发生时间 |

## 三、事件来源

| 来源 | 触发场景 |
|------|---------|
| 告警中心 | 告警创建、认领、静默、关闭 |
| 任务中心 | 任务执行、完成、失败 |
| 应用发布 | 部署开始、成功、回滚 |
| AIOps 智能体 | 工具调用、动作确认 |
| Zabbix 告警导入 | Zabbix 问题导入为告警 |
| CMDB | CI 创建、修改、删除 |

## 四、筛选与分析

- 按系统、环境、应用、时间范围筛选
- 按模块和类别分组
- 查看关联事件链（父子事件关系）
- 故障分析视图：自动关联时间窗口内的异常事件

## 五、事件源管理

"事件源"Tab 管理外部事件源的接入配置，支持将通过 API 推送的自定义事件纳入事件墙。
