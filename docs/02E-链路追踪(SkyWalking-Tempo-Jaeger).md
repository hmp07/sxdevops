# SxDevOps 链路追踪 (SkyWalking / Tempo / Jaeger)

## 一、概述

链路追踪模块支持多种 APM 后端，用于分布式调用链分析。

**支持的后端：**
- Apache SkyWalking
- Grafana Tempo
- Jaeger
- Zipkin

**权限：** `ops.trace.view`

## 二、数据源配置

侧边栏：**可观测性 → 数据源 → 链路追踪数据源** Tab

| 字段 | 说明 |
|------|------|
| 数据源名称 | 自定义 |
| 追踪类型 | SkyWalking / Tempo / Jaeger / Zipkin |
| API 地址 | 如 SkyWalking OAP `http://oap:12800` |
| 认证信息 | 根据后端类型填写 |

## 三、使用

侧边栏：**可观测性 → 链路追踪**

- 选择数据源
- 按服务名、Trace ID、时间范围搜索
- 查看调用链拓扑图和各 Span 详情
- 支持与日志关联跳转

## 四、关联配置

在**关联配置**中维护日志→链路→仪表盘的跳转映射，实现三向关联导航。配置项：

| 字段 | 说明 |
|------|------|
| 关联名称 | 如 `生产-Loki→Tempo` |
| 日志数据源 | 选择已配置的日志数据源 |
| 链路数据源 | 选择已配置的追踪数据源 |
| 启用 / 默认 | 状态控制 |
