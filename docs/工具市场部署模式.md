# 工具市场部署模式说明

工具市场现在支持两种部署方式：

- `Docker Compose 单机`
- `Kubernetes`

同一份服务模板可以同时声明这两种能力，页面会根据模板自动展示可选部署模式。

## 1. Docker Compose 单机模式

适用场景：

- 演示环境
- 单机中间件
- 远程 Linux 主机快速安装

执行流程：

1. 在工具市场选择服务模板
2. 选择 `Docker Compose 单机`
3. 选择目标主机
4. 填写版本与参数
5. 平台通过 SSH 上传 `docker-compose.yml`
6. 在远端执行 `docker-compose up -d` 或 `docker compose up -d`

支持动作：

- 发起部署
- 查看部署日志
- 停止服务
- 启动服务
- 卸载服务

## 2. Kubernetes 模式

适用场景：

- 测试 / 预发 / 生产集群
- 需要命名空间隔离
- 需要副本数控制
- 希望统一纳管到 K8s

执行流程：

1. 在工具市场选择服务模板
2. 选择 `Kubernetes`
3. 选择目标集群
4. 填写命名空间、发布名称、副本数、版本和模板参数
5. 平台渲染 K8s YAML
6. 自动补充 `managed-by / instance / name` 标签
7. 自动确保目标命名空间存在
8. 调用 Kubernetes API 下发资源

支持动作：

- 发起部署
- 查看运行日志
- 停止服务（缩容到 0）
- 启动服务（恢复模板副本数）
- 卸载服务（删除平台管理的 K8s 资源）

## 3. 目标对象说明

两种模式的目标对象不同：

- `Docker Compose 单机`：目标为 `Host`
- `Kubernetes`：目标为 `K8sCluster`

部署记录展示也会随模式变化：

- Docker 模式显示主机名和 IP
- K8s 模式显示集群名和命名空间

## 4. 升级与数据迁移

本次升级已包含数据迁移，执行数据库迁移后会自动完成以下处理：

- 为历史工具市场模板补齐 `k8s_manifest_template`
- 为历史部署记录补齐 `deploy_mode`
- 为历史部署记录补齐 `release_name`
- 为历史部署记录补齐 `replicas`

执行命令：

```bash
cd backend
python manage.py migrate
```

## 5. 模板初始化 / 刷新

如果你想把内置模板刷新到最新版本，执行：

```bash
cd backend
python manage.py seed_templates
```

执行后会刷新内置 17 个模板的模板内容，包括 K8s YAML：

- MySQL
- Redis
- PostgreSQL
- MongoDB
- Nginx
- Jenkins
- GitLab
- Grafana
- Elasticsearch
- Loki
- JumpServer
- Nacos
- XXL-Job
- Java
- Python
- Go
- Node.js

## 6. 使用建议

- 本地演示优先使用 `Docker Compose 单机`
- 团队测试环境和共享环境优先使用 `Kubernetes`
- 生产环境建议先审阅模板 YAML，再由受控账号发起部署
- 语言运行环境模板更适合开发 / 调试容器基座，如需业务端口可按团队标准继续扩展模板
- 如需更贴近生产的存储、Ingress、资源限制或探针策略，可继续扩展模板内容
