# SxDevOps 资源底座（主机分组管理）

## 一、概述

资源底座是任务中心的数据基础，统一管理所有可操作的主机资源及其分组。

**入口：** 侧边栏 → 任务中心 → 资源底座
**权限：** `ops.task.resource.view` / `ops.task.resource.manage`

## 二、主机管理

### 2.1 添加主机

| 字段 | 说明 |
|------|------|
| 主机名 | 主机标识 |
| IP 地址 | 管理 IP |
| SSH 端口 | SSH 端口（默认 22） |
| SSH 用户 | 登录用户名 |
| SSH 密码 / 密钥 | 认证方式 |
| 业务线 | 所属业务系统 |
| 环境 | 生产/测试/开发 |

### 2.2 主机分组

将主机按业务逻辑分组（如"生产-Web 服务器组"），任务执行时可按组选择目标。

### 2.3 资源类型

资源底座不仅管理物理主机，还包括：
- K8s 集群节点
- Docker 主机
- 云主机（通过云 SDK 同步）

## 三、与任务工作台联动

资源底座中定义的主机分组直接出现在任务工作台的"选择目标"列表中。管理员在资源底座维护分组，运维人员在工作台中直接使用。
