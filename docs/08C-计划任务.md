# SxDevOps 计划任务

## 一、概述

计划任务支持按 cron 表达式定时执行运维操作——巡检、备份、清理等周期性工作。

**入口：** 侧边栏 → 任务中心 → 计划任务（Feature Flag 控制，默认关闭）
**权限：** `ops.host.schedule.view` / `ops.host.schedule.manage` / `ops.host.schedule.execute`

## 二、启用计划任务

计划任务功能默认关闭（`TASK_SCHEDULES_VISIBLE = false` 在 `frontend/src/router/index.js`）。开启方法：

1. 将 `TASK_SCHEDULES_VISIBLE` 设为 `true`
2. 重新构建前端

## 三、创建计划任务

| 字段 | 说明 |
|------|------|
| 任务名称 | 自定义 |
| cron 表达式 | 如 `0 2 * * *`（每天凌晨 2 点） |
| 任务类型 | 命令执行 / 脚本模板 |
| 目标主机 | 选择主机或主机组 |
| 命令/脚本 | 具体执行内容 |

## 四、Cron 表达式

```
分 时 日 月 周
*  *  *  *  *
```

| 表达式 | 含义 |
|--------|------|
| `*/5 * * * *` | 每 5 分钟 |
| `0 * * * *` | 每小时整点 |
| `0 2 * * *` | 每天凌晨 2 点 |
| `0 9 * * 1-5` | 工作日早上 9 点 |

## 五、执行历史

与任务工作台共享执行历史。计划任务自动生成的执行记录标记来源为 `schedule`，方便区分手动执行和自动执行。
