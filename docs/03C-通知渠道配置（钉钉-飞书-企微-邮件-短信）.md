# SxDevOps 通知渠道配置

## 一、概述

告警通知支持钉钉、飞书、企业微信、邮件、短信、语音等多种渠道。通知规则决定哪些告警通过哪些渠道发送给哪些人。

**入口：** 侧边栏 → 告警中心 → 通知配置 Tab
**权限：** `ops.alert.notify` / `ops.alert.config.view`

## 二、通知渠道

### 2.1 支持的渠道类型

| 渠道 | 配置方式 | 说明 |
|------|---------|------|
| 钉钉 (DingTalk) | Webhook URL 或 Access Token | 支持卡片交互按钮 |
| 飞书 (Feishu) | Webhook URL | 支持交互式卡片 |
| 企业微信 (WeCom) | Webhook URL 或 Key | Markdown 格式通知 |
| 邮件 | SMTP 配置（Django 邮件后端） | 支持模板渲染 |
| 短信 | Webhook HTTP 代理 | 需第三方短信网关 |
| 语音 | Webhook HTTP 代理 | 需第三方语音网关 |

### 2.2 创建通知渠道

| 字段 | 说明 |
|------|------|
| 渠道名称 | 如 `钉钉-运维告警群` |
| 渠道类型 | 选择类型 |
| Webhook URL | 群机器人 Webhook 地址 |
| Access Token | 或使用 Token 方式 |
| 通知模板标题 | 支持变量替换 |
| 通知模板正文 | 支持 Markdown 和变量 |
| 发送恢复通知 | 告警恢复时是否通知 |
| 启用 | 控制渠道状态 |

> 创建后可点击"测试"发送测试消息验证配置。

## 三、通知接收人

### 3.1 接收人

| 字段 | 说明 |
|------|------|
| 姓名 | 显示名称 |
| 手机号 | 用于短信/语音 |
| 邮箱 | 用于邮件通知 |
| 平台用户 | 关联平台账号 |
| 钉钉/飞书/企微 ID | 用于精准推送 |

### 3.2 接收组

将多个接收人分组，如"运维值班组"、"DBA 组"。可包含接收人和平台用户。

## 四、通知规则

通知规则连接"什么告警 → 什么渠道 → 通知谁"：

| 配置项 | 说明 |
|--------|------|
| 名称 | 规则名称 |
| 匹配条件 | 筛选适用此规则的告警 |
| 最低级别 | 只有 >= 此级别的告警才触发通知 |
| 通知渠道 | 选择一个或多个渠道 |
| 接收组 / 接收人 | 通知目标 |
| 聚合规则 | 关联的聚合规则（避免重复通知） |
| 升级策略 | 关联的升级策略 |
| 触发时机 | 告警触发 / 恢复 / 升级时分别是否通知 |

## 五、通知记录

通知配置 Tab 下的"通知记录"子 Tab 展示所有已发送通知的历史：
- 发送时间、动作（fire/resolved/escalation）
- 渠道、规则、接收人
- 状态（成功 / 跳过 / 错误）
- 错误信息（如发送失败的原因）

## 六、变量模板

通知标题和正文支持以下变量（`{变量名}` 格式）：

| 变量 | 说明 |
|------|------|
| `{title}` | 告警标题 |
| `{level}` | 告警级别 |
| `{status}` | 告警状态 |
| `{source}` | 告警来源 |
| `{source_type}` | 来源类型 |
| `{service}` | 关联服务 |
| `{environment}` | 环境 |
| `{resource}` | 资源标识 |
| `{message}` | 告警详情 |
| `{runbook_url}` | Runbook 链接 |
| `{starts_at}` | 触发时间 |
| `{group_key}` | 聚合键 |
| `{occurrence_count}` | 出现次数 |
