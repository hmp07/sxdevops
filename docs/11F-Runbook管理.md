# SxDevOps Runbook 管理

## 一、概述

Runbook 是标准运维操作流程的电子化版本，指导运维人员按步骤处理特定场景。AIOps 智能助手可以引用 Runbook 内容辅助排障。

**入口：** AIOps → 智能体配置 → Runbook 管理
**权限：** `aiops.runbook.view` / `aiops.runbook.manage`

## 二、Runbook 结构

| 字段 | 说明 |
|------|------|
| 名称 | Runbook 标题 |
| 分类 | 故障处理 / 巡检 / 变更 / 应急 |
| 适用场景 | 触发条件和前置条件 |
| 步骤列表 | 编排的操作步骤（每步含动作、预期结果、回滚方案） |
| 关联资源 | Runbook 适用的服务/系统/环境 |
| 版本 | 版本号 |

## 三、使用场景

| 场景 | Runbook 作用 |
|------|------------|
| AI 排障 | AI 助手分析问题时引用相关 Runbook 的操作建议 |
| 人工参考 | 运维人员在事件中心查看告警时，关联 Runbook 一键打开 |
| 自动化触发 | 特定告警触发时可建议执行对应 Runbook |

## 四、与告警关联

告警的 `runbook_url` 字段存储关联的 Runbook 链接。告警触发时，通知消息中自动附上 Runbook 链接，接收人可直接跳转到操作指南。
