# SxDevOps 知识图谱

## 一、概述

知识图谱将平台接入的各类资源（主机、容器、CMDB 配置项、Zabbix 数据源、iTop CI、K8s 集群）建模为节点和边，提供统一的拓扑视图和环境感知能力。

**入口：** 侧边栏 → AIOps → 知识图谱
**权限：** `aiops.knowledge.view`

## 二、图谱构成

### 2.1 节点类型

| 节点类型 | 来源 |
|---------|------|
| 功能节点 | 可观测性、事件中心、任务中心、CMDB 等平台能力 |
| 数据源节点 | Zabbix Server、iTop Server 的接入配置 |
| 环境节点 | 生产、测试等知识环境 |
| 资源节点 | 主机、K8s 集群、Docker 主机 |
| CI 节点 | CMDB 配置项 |

### 2.2 边类型

| 边类型 | 含义 |
|--------|------|
| `capability_datasource` | 功能 → 数据源：该功能由哪个外部系统提供数据 |
| `zabbix_monitored` | Zabbix 监控的主机 |
| `zabbix_triggered` | Zabbix 告警触发 |
| `deployed_on` | 应用部署在主机上 |
| `depends_on` | CI 之间的依赖关系 |

### 2.3 环境筛选

知识环境 (`KnowledgeEnvironment`) 允许限定图谱显示范围——只展示特定环境、特定数据源、特定 K8s 命名空间下的资源。

## 三、交互

- 点击节点展开关联
- 拖拽调整布局
- 环境切换过滤视图
- 节点详情显示资源统计（如 Zabbix 数据源节点显示关联的主机数）

## 四、数据来源

知识图谱数据来自：
- `ZabbixDataSource` 表 → Zabbix 数据源节点
- `iTopDataSource` 表 → iTop 数据源节点
- `DeviceMapping` 表 → Zabbix 监控边
- `ConfigItem` + `CIRelation` 表 → CI 节点和边
- `K8sCluster` + `DockerHost` 表 → 资源节点
