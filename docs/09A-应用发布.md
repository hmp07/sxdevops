# SxDevOps 应用发布

## 一、概述

应用发布模块管理从开发到生产的部署流程，支持 Docker Compose 和 Kubernetes 两种部署模式，以及多级审批流。

**入口：** 侧边栏 → 工单系统 → 应用发布
**权限：** `ops.deployment.view` / `ops.deployment.manage` / `ops.deployment.approve`

## 二、发布模式

| 模式 | 说明 |
|------|------|
| Docker Compose | 在指定的 Docker 主机上执行 docker-compose |
| Kubernetes | 部署到 K8s 集群的指定 namespace |

## 三、发布流程

### 3.1 创建发布

1. 选择发布模式（Docker Compose / K8s）
2. 选择目标主机或集群
3. 填写应用信息（业务线、应用名、环境、版本号）
4. 上传配置（docker-compose.yml / K8s YAML）
5. 提交审批

### 3.2 审批流程

- 发布提交后进入审批流
- 审批人可查看变更摘要
- 支持多级审批
- 审批通过后自动执行

### 3.3 执行与回滚

- 实时查看部署日志
- 部署成功后记录当前版本
- 支持回滚到上一版本

## 四、发布历史

"应用发布"展示当前所有部署的状态，"审批流"Tab 展示审批记录。每次发布关联事件墙，形成完整的变更审计链。
