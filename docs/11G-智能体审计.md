# SxDevOps 智能体审计

## 一、概述

智能体审计记录 AI 智能体的所有活动——会话、工具调用、模型调用、动作执行和成本——确保 AI 行为可追溯、可审查。

**入口：** 侧边栏 → AIOps → 智能体审计
**权限：** `aiops.audit.view`

## 二、审计维度

### 2.1 会话审计

| 字段 | 说明 |
|------|------|
| 会话 ID | 每次对话的唯一标识 |
| 用户 | 发起对话的用户 |
| 开始/结束时间 | 会话时间范围 |
| 消息数 | 对话轮次 |
| 模型 | 使用的 LLM 供应商和模型 |

### 2.2 工具调用审计

| 字段 | 说明 |
|------|------|
| 工具名称 | MCP 工具标识 |
| 调用时间 | 调用发生时间 |
| 成功/失败 | 调用结果 |
| 耗时 | 执行耗时 |
| 参数摘要 | 调用参数的摘要（脱敏后） |

### 2.3 模型调用审计

| 字段 | 说明 |
|------|------|
| 请求模型 | `requested_model` vs 实际使用的 `resolved_model` |
| Token 用量 | prompt_tokens + completion_tokens |
| 耗时 | 模型响应延迟 |
| 费用 | 估算的调用成本 |

### 2.4 Skill 追踪

记录 AI 在对话中匹配和使用了哪些 Skill 包，以及 Skill 输出是否符合合同规范。

### 2.5 成本统计

按时间范围汇总：
- 总调用次数
- 总 Token 用量
- 总费用（按供应商和货币拆分）
- 趋势图

## 三、费用计算

```
费用 = (prompt_tokens × 输入单价 + completion_tokens × 输出单价) / 1,000,000
```

单价在模型供应商配置中设定。支持 USD 和 CNY 两种货币。

## 四、数据保留

审计日志是持久数据，不会自动清理。建议定期归档历史数据。
