# SxDevOps 告警事件管理

## 一、概述

告警事件管理是告警中心的核心操作界面，用于查看、筛选、认领、静默和关闭来自不同数据源的告警。

**入口：** 侧边栏 → 可观测性 → 告警中心 → 告警事件 Tab
**权限：** `ops.alert.view` / `ops.alert.manage`

## 二、告警列表

### 2.1 统计卡片

| 卡片 | 点击效果 |
|------|---------|
| 全部告警 | 清除筛选 |
| 活跃告警 | 筛选 status=active |
| 严重告警 | 筛选 level=critical |
| 已屏蔽告警 | 筛选 status=muted |

### 2.2 筛选器

| 筛选维度 | 选项 |
|---------|------|
| 状态 | 活跃 / 已恢复 / 已屏蔽 / 已关闭 |
| 来源 | Prometheus / Zabbix / 夜莺 / 阿里云 / 通用 Webhook |
| 环境 | 从告警数据中动态提取 + 支持自定义输入 |
| 级别 | 严重 / 警告 / 信息 |
| 搜索 | 标题 / 来源 / 服务 / 资源模糊匹配 |

### 2.3 列表模式 vs 分组模式

- **列表模式**：按时间排序的告警列表，每行显示关键字段和操作按钮
- **分组模式**：按维度（source_type、environment、service 等）聚合，每组显示总数和各级别计数

## 三、告警操作

| 操作 | 说明 | 权限 |
|------|------|------|
| 认领 (Claim) | 标记自己为处理人 | `ops.alert.manage` |
| 取消认领 | 释放认领状态 | `ops.alert.manage` |
| 确认 | 标记告警已确认 | `ops.alert.manage` |
| 静默 (Mute) | 临时屏蔽告警通知（默认 1 小时） | `ops.alert.manage` |
| 关闭 | 手动关闭告警 | `ops.alert.manage` |
| 详情 | 查看告警完整信息和操作时间线 | `ops.alert.view` |

## 四、告警详情抽屉

点击"详情"打开右侧抽屉，显示：
- 告警基本信息（标题、级别、状态、来源）
- 指纹和聚合键
- 关联资源、服务、环境
- 标签和注解（JSON KV）
- 原始 Payload（方便排障）
- 操作时间线（创建、认领、静默、通知等历史动作）
