# SxDevOps 任务工作台（批量命令与脚本模板）

## 一、概述

任务工作台是运维操作的执行中心，支持对主机执行批量命令、脚本模板、任务草稿和执行历史查看。

**入口：** 侧边栏 → 任务中心 → 任务工作台
**权限：** `ops.task.execute` / `ops.host.execute`

## 二、任务类型

| 类型 | 说明 |
|------|------|
| 批量命令 | 对选中的主机执行 Shell / PowerShell 命令 |
| 脚本模板 | 使用预定义模板执行标准化运维操作 |
| K8s Pod 操作 | 重启 Pod、执行命令、扩缩容 |
| 巡检任务 | 对主机执行定期健康检查 |

## 三、使用流程

### 3.1 发起任务

1. 点击"新建任务"
2. 选择任务类型（命令 / 脚本 / K8s）
3. 选择目标主机（从资源底座中选择）
4. 填写命令内容或选择脚本模板
5. 提交执行

### 3.2 查看执行结果

- 实时查看执行日志
- 成功/失败统计
- 每台主机的输出内容

### 3.3 任务草稿

未提交的任务自动保存为草稿，下次打开恢复编辑状态。

## 四、脚本模板

预定义的运维操作模板，确保标准化执行：
- 模板名称和分类
- 脚本内容（支持变量替换）
- 目标主机类型限制

## 五、执行历史

所有已执行的任务保存在执行历史中，支持按时间、状态、主机筛选。
