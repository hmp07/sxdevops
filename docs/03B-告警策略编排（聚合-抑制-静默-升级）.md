# SxDevOps 告警策略编排

## 一、概述

告警策略编排提供四种规则类型，用于自动处理告警事件：聚合、抑制、静默和升级。所有规则通过标签匹配器（Matcher）筛选适用的告警。

**入口：** 侧边栏 → 告警中心 → 策略编排 Tab
**权限：** `ops.alert.config.view` / `ops.alert.config.manage`

## 二、聚合规则 (Aggregation)

将相似告警按维度聚合，减少重复通知。

| 配置项 | 说明 |
|--------|------|
| 匹配条件 | 筛选哪些告警需要聚合 |
| 聚合维度 | 按哪些字段分组（source_type, environment, service 等） |
| 窗口时间 | 聚合时间窗口 |
| 重复间隔 | 同一聚合组最近一次通知后的静默间隔 |

## 三、抑制规则 (Inhibition)

当某些告警（源）触发时，自动抑制另一些告警（目标）的通知。

| 配置项 | 说明 |
|--------|------|
| 源匹配条件 | 触发抑制的告警条件 |
| 目标匹配条件 | 被抑制的告警条件 |
| 等值标签 | 源和目标必须相同的标签 |
| 抑制时长 | 抑制持续时间 |

## 四、静默规则 (Mute)

按时间窗口批量屏蔽告警通知，用于计划内维护。

| 配置项 | 说明 |
|--------|------|
| 匹配条件 | 筛选需要静默的告警 |
| 开始/结束时间 | 静默时间窗口 |
| 原因 | 静默原因说明 |

## 五、升级策略 (Escalation)

告警持续未处理时自动升级通知级别。

| 配置项 | 说明 |
|--------|------|
| 匹配条件 | 筛选需要升级的告警 |
| 升级级别 | 多级定义（每级含名称、等待时间、通知渠道） |
| 重复间隔 | 升级通知的最小间隔 |

## 六、匹配器语法

匹配器使用 `key op value` 三元组筛选告警：

| 操作符 | 含义 | 示例 |
|--------|------|------|
| `==` | 等于 | `source_type == prometheus` |
| `!=` | 不等于 | `level != info` |
| `=~` | 正则匹配 | `service =~ order-.*` |
| `!~` | 正则不匹配 | `resource !~ test-.*` |
| `in` | 属于列表 | `environment in [prod, staging]` |
| `not in` | 不属于列表 | `severity not in [info]` |
| `contains` | 包含子串 | `title contains 磁盘` |
