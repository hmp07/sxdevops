# SxDevOps 事务工单

## 一、概述

事务工单用于记录和跟踪非发布类的运维事务——如权限申请、资源变更、问题处理等。

**入口：** 侧边栏 → 工单系统 → 事务工单
**权限：** `ops.ticket.view` / `ops.ticket.manage` / `ops.ticket.approve`

## 二、工单类型

| 类型 | 说明 |
|------|------|
| 变更工单 | IT 基础设施变更 |
| 事件工单 | 故障处理和问题跟踪 |
| 服务请求 | 用户服务申请 |

从 iTop 同步的工单（UserRequest、Incident、NormalChange、Problem）自动归类到对应类型。

## 三、工单生命周期

```
新建 → 待审批 → 处理中 → 已完成
                    ↓
                  驳回 → 已关闭
```

### 3.1 创建工单

| 字段 | 说明 |
|------|------|
| 工单标题 | 简要描述 |
| 工单类型 | 变更 / 事件 / 服务请求 |
| 详情 | 工单详细描述 |
| 优先级 | 低 / 中 / 高 / 紧急 |
| 指派人 | 处理人 |

### 3.2 审批

- 审批人可查看工单详情
- 通过 / 驳回
- 驳回时需填写原因
- 审批状态可跟踪

## 四、与 iTop 联动

从 iTop 同步的工单（`external_source='itop'`）：
- 显示在事务工单列表中
- 标记来源为 `iTop`
- 可通过平台操作后反向同步（如关闭工单）
