# SxDevOps 模块开关管理

## 一、概述

模块开关允许管理员按需启用或停用整个功能模块。关闭模块后，前端菜单和路由对所有人不可见（包括管理员），但后端 API 和数据库不受影响。

**入口：** 侧边栏 → 系统管理 → 模块管理
**权限：** `rbac.module.manage`

## 二、可开关的模块

| 模块 | 编码 | 说明 |
|------|------|------|
| 仪表盘 | `dashboard` | 运行概览首页（必开启） |
| AIOps | `aiops` | 智能助手、知识图谱（必开启） |
| 可观测性 | `observability` | 指标、日志、链路、告警（必开启） |
| 事件中心 | `events` | 事件墙、事件源 |
| 任务中心 | `tasks` | 任务工作台、资源底座 |
| 工单系统 | `workorders` | 发布、SQL 审计、事务工单 |
| 容器管理 | `containers` | K8s、Docker |
| 系统管理 | `system` | 用户、审计、模块管理（必开启） |

标记为"必开启"的模块不可停用——它们是平台运行的核心。

## 三、操作

- 切换开关 → 立即生效（前端刷新后）
- 已停用模块的菜单项消失
- 直接访问已停用模块的 URL → 跳转到首页

## 四、与权限的关系

模块开关优先级高于 RBAC 权限：即使拥有某模块的权限，如果模块被关闭，也无法访问。这主要用于大型部署中按需裁剪功能。
