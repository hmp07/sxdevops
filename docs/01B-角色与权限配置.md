# SxDevOps 角色与权限配置

## 一、概述

SxDevOps 的 RBAC 系统覆盖三层权限控制：后端 API、前端路由/菜单、页面内操作按钮。权限定义在 `backend/rbac/registry.py`，前端仅是镜像，**后端才是安全边界**。

## 二、权限体系

### 2.1 权限编码规范

采用 `模块.资源.操作` 三级命名：

```
模块         .  资源        .  操作
ops          .  host        .  view
aiops        .  chat        .  analyze
cmdb         .  ci          .  manage
rbac         .  user        .  manage
```

### 2.2 内置角色

| 角色 | 编码 | 说明 |
|------|------|------|
| 平台管理员 | `platform-admin` | 全部权限，超级用户 |
| 运维工程师 | `ops-admin` | 运维操作权限（主机/任务/告警/容器/CMDB/Zabbix） |
| 开发工程师 | `dev-user` | 可观测性查看 + 工单提交 |
| 审计员 | `auditor` | 只读审计权限 |
| 观察者 | `viewer` | 最小只读权限 |

### 2.3 权限分组（模块维度）

| 模块 | 权限数量 | 典型权限 |
|------|---------|---------|
| `ops` | ~50+ | dashboard、host、task、deployment、alert、k8s、docker、zabbix、grafana、tracing、metric |
| `aiops` | ~15 | chat.view、chat.analyze、config、knowledge、mcp、skill、runbook、audit |
| `cmdb` | ~10 | dashboard.view、ci.view、ci.manage、topology、itop、request、cost |
| `eventwall` | ~4 | view、source.view、source.manage |
| `rbac` | ~8 | user.view、user.manage、role、group、audit、module.manage |
| `sqlaudit` | ~8 | datasource、order、query |

## 三、配置操作

### 3.1 查看权限列表

侧边栏：**系统管理 → 用户管理 → 权限管理** Tab

### 3.2 创建/编辑角色

1. **系统管理 → 用户管理 → 角色管理** Tab
2. 新建角色或编辑现有角色
3. 选择角色的权限集合
4. 保存

### 3.3 为用户分配角色

1. **系统管理 → 用户管理 → 用户列表** Tab
2. 编辑用户
3. 在"角色"下拉框中选择一个或多个角色
4. 保存后用户重新登录生效（或点击"刷新权限"）

## 四、新增模块的 RBAC 接入步骤

遵循以下 5 步流程确保权限完整：

### Step 1: 注册权限码

`backend/rbac/registry.py`：

```python
PERMISSION_DEFINITIONS = [
    # ...
    ('ops.new_module.view', '查看新模块', 'ops', '查看新模块的只读权限'),
    ('ops.new_module.manage', '管理新模块', 'ops', '管理新模块的写入权限'),
]
```

### Step 2: 后端视图加入权限检查

```python
from rbac.permissions import build_rbac_permission

class NewModuleViewSet(RBACPermissionMixin, viewsets.ModelViewSet):
    rbac_permissions = {
        'list': 'ops.new_module.view',
        'create': 'ops.new_module.manage',
        'update': 'ops.new_module.manage',
        'destroy': 'ops.new_module.manage',
    }
```

### Step 3: 注册 URL

`backend/<app>/urls.py` — 路由自动继承 ViewSet 的权限。

### Step 4: 添加前端路由守卫

`frontend/src/router/index.js`：

```javascript
{
  path: 'new-module',
  component: () => import('@/views/NewModule.vue'),
  meta: { permission: 'ops.new_module.view' },  // 或 anyPermissions
}
```

### Step 5: 添加菜单可见性

`frontend/src/layout/AppLayout.vue`：

```javascript
{
  path: '/new-module',
  title: '新模块',
  icon: 'Setting',
  permission: 'ops.new_module.view',
}
```

### Step 6: 重新同步权限

```bash
cd backend && python manage.py seed_data   # 或调用 API 同步
# POST /api/auth/sync/
```

## 五、超级用户

`is_superuser=True` 的用户自动拥有所有权限，所有 `hasPermission()` / `hasAnyPermission()` 调用对其返回 `true`。无需为其分配角色。

## 六、模块开关

系统管理 → 模块管理 可启用/禁用整个功能模块。模块关闭后：
- 侧边栏菜单不显示该模块
- 即使有权限也无法访问对应路由
- 后端 API 仍然受权限保护（不会被前端开关绕过）

## 七、验证权限

1. 使用不同角色账号登录（如 `viewer_demo`）
2. 确认侧边栏只显示对应权限的菜单
3. 尝试直接访问无权限的 URL → 应跳转 403 页面
4. 使用 admin 登录 → 应看到全部菜单和功能

## 八、权限同步

新增权限码后，运行以下命令同步到数据库：

```bash
cd backend && python manage.py seed_data
```

或通过 API：`POST /api/auth/sync/`（需要 `rbac.permission.manage` 权限）。
