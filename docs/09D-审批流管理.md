# SxDevOps 审批流管理

## 一、概述

审批流为关键操作提供多级审批机制，确保变更操作有适当的授权。目前应用于应用发布和事务工单。

**入口：** 侧边栏 → 工单系统 → 审批流
**权限：** `ops.deployment.approve` / `ops.ticket.approve`

## 二、审批流类型

| 类型 | 触发场景 |
|------|---------|
| 发布审批 | 应用发布提交后 |
| 工单审批 | 事务工单提交后 |
| SQL 审批 | SQL 工单提交后 |

## 三、审批流程

### 3.1 提交审批

发布/工单创建后自动进入审批流。审批人收到通知（通知铃 + 邮件/IM）。

### 3.2 审批操作

| 操作 | 说明 |
|------|------|
| 通过 | 进入下一级审批或直接执行 |
| 驳回 | 退回提交人，需修改后重新提交 |
| 加签 | 增加额外审批人 |

### 3.3 多级审批

支持多级审批链，每级可以有多个审批人（任一通过即可）。所有级别的审批记录都保存。

## 四、审批历史

每个发布/工单的审批流程全程留痕，可查看：
- 每级审批人和审批时间
- 审批意见
- 最终结果

审批记录同步写入事件墙，形成完整的变更审计链。
