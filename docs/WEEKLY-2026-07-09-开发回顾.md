## 一、核心成果：CMDB 数据融入 AIOps 知识图谱与智能助手

### 1.1 知识图谱数据集成

**改动文件**：`backend/aiops/knowledge_graph.py`（+260 行）

- CMDB CI 数据（来自 iTop 同步）自动加载为知识图谱节点
  - ApplicationSolution → `system` 节点（业务系统）
  - BusinessProcess → `service` 节点
  - DBServer/WebServer/WebApplication → `runtime_component` 节点
  - Server/VirtualMachine → `infrastructure` 节点
- CIRelation 边映射为知识图谱关系
- BFS 双向传播 business_line，从 ApplicationSolution 沿拓扑链传递到所有下游 CI
- Zabbix DeviceMapping 双向去重：通过 IP + 主机名匹配，将 TaskResource 基础设施节点与 CMDB 元数据富化合并，避免重复节点
- 知识图谱从 22 节点增长到 53 节点、96 条边

### 1.2 工具系统建设

**改动文件**：`backend/aiops/services.py`（+448 行）

- 新增 2 个内置工具：
  - `query_cmdb_items` — 按业务线/关键词搜索 CMDB 配置项
  - `query_cmdb_topology` — BFS 拓扑查询，结合 Zabbix 监控状态
- 新增 `sx-cmdb-query` Skill（CMDB 查询 SOP）
- 新增 `cmdb.query` Action（Action Router 匹配）
- 集成到知识图谱 MCP Server 的 tool_whitelist

### 1.3 工具系统全面审计与修复

经过 3 个 agent 并行扫描，发现了工具系统的系统性脆弱性：新增一个工具需要在 7 个独立位置注册。本次修复了：

| 类别 | 数量 | 说明 |
|------|------|------|
| Bug 修复 | 7 项 | 缺参数、N+1 查询、变量不一致、JSON 注入、全表扫描、统计遗漏、get_or_create 重复 |
| 死代码清理 | 3 项 | 未使用导入、死参数、未使用计算属性 |
| 样式修复 | 3 项 | 重复 CSS、重复 label key、IME 输入法拦截 |

### 1.4 DeepSeek 模型兼容性

- 修复 `_model_request_payload_variants()`：非 GPT-5 模型不再包含 `developer_payload` 变体
- 修复 `tool_compatible_payload` 构建：使用原始 `system` 角色消息替代 `developer` 角色
- 添加 `tool` 响应消息配对保证，满足 DeepSeek 严格的消息验证
