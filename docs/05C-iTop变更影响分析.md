# SxDevOps iTop 变更影响分析

## 一、概述

利用 iTop CMDB 的 CI 关联关系，结合 Zabbix 监控和平台告警数据，AIOps 智能助手可以分析变更的潜在影响范围。

**前置条件：**
- 已配置 iTop 数据源并执行过同步
- 已配置 Zabbix 数据源（可选，增强分析）
- 已配置模型供应商（LLM）

## 二、触发方式

在 AI 智能助手中用自然语言发起：

```
"这个变更会影响哪些服务器"
"分析订单服务的变更对下游系统的影响"
"查下 iTop 中这个 CI 关联了哪些设备"
```

## 三、分析维度

AI 助手通过以下工具链进行分析：

### 3.1 CI 关系追踪

调用 iTop MCP 工具：
- `itop_get_cis`：查询目标 CI
- `itop_get_related`：追溯关联关系（支持深度和方向配置）

### 3.2 监控状态关联

查询 Zabbix 主机状态和当前问题，判断受影响设备是否存在已知异常。

### 3.3 告警关联

查询平台告警中心中与目标 CI 相关的活跃告警。

### 3.4 变更历史

查询近期变更记录和事件墙，关联时间窗口内的异常事件。

## 四、分析输出

AI 生成结构化分析报告：

- **影响范围清单**：直接和间接受影响的 CI 列表
- **风险提示**：当前有异常告警的设备
- **建议顺序**：推荐的变更窗口和回滚方案

## 五、相关动作

分析完成后，AI 可建议生成工单：
- 创建事务工单记录分析结果
- 生成变更通知给相关人员

所有分析结果记录在 AIOps 审计日志中。
