# CMDB (配置管理数据库) 使用手册

## 1. 模块概述
CMDB (Configuration Management Database) 是本运维平台的核心基础设施。它集中管理了企业内所有的数字资产（如云主机、数据库、容器集群、网络组件及微服务等），并负责维护这些资产之间错综复杂的拓扑依赖关系。

---

## 2. 核心功能操作指南

### 2.1 资源配置管理 (列表视图)
基于日常运维习惯，CMDB 采用**分层级、树状过滤**的管理模式。

*   **业务资源树过滤**：
    在页面左侧，我们提供了“业务线 -> 环境”的层级结构树。您可以通过点击相关的节点（例如：`电商线`下的`生产环境`），快速筛选并查看该分组下的所有云资源。
    > [!TIP]
    > 点击最上方的“业务资源树”标题，会清空所有过滤条件，恢复展示全量数据。
*   **动态数据统计卡片**：
    页面顶部的资源总数、状态分布及各类型的卡片统计，会与您的过滤条件（搜索词、所选环境、所选状态等）**保持 100% 联动实时计算**。
*   **属性动态扩展**：
    每一项资源都支持记录详细的硬件规格（CPU/内存/磁盘）、内网IP、拥有者以及**月度开销成本**信息。

### 2.2 资源地图 (拓扑视图)
资源地图利用可视化 Canvas 技术，将原本枯燥的数据转化为连接成网的系统架构图。支持节点拖拽、鼠标悬浮展示详情以及自动分层排布。

#### 核心特性与高级筛选逻辑
为了保证大规模数据集下的视图清晰度，我们在地图上方提供了三个级联筛选下拉框：

1.  **环境级别的绝对隔离**：
    当您在“按环境筛选”中选择了某个环境（如 `测试环境`）时，后端的拓扑计算引擎会自动切断跨环境之间的非安全连线，**保证画布上只会出现当前环境的合法节点**。
2.  **“一级节点”连线延伸过滤（特色功能）**：
    在“按类型筛选”中，如果您指定了某个具体类型（例如 `MySQL`），系统会以满足条件的节点为辐射中心，**自动向外延伸展示与之直接关联的第一层（1-hop）资源**。

#### 应用场景与案例分析
这一机制极大地增强了以具体组件为核心的排障与架构盘点能力：

*   **案例一：数据库爆炸半径分析**
    当您查询某个**数据库**时，所有与其有读写连接关系的**微服务程序**都会围绕在它身边显示出来。
    *   *价值*：运维人员准备对某数据库进行维护时，可以一目了然地知道即将影响哪些业务，提前发出精准通知，避免误伤。
*   **案例二：主机架构盘点**
    当您查询某台**云主机**时，运行在它上面的所有**服务与组件**也会跟随它一并出现在地图上。
    *   *价值*：非常适合做链路排障和以单独节点为中心的架构盘点，清晰展现主机的角色定位。
*   **案例三：故障影响面评估**
    在拓扑中选择核心 Redis，由于一级连线延伸机制，所有依赖该 Redis 的上游业务都会浮现，帮助快速定位故障波及范围。

#### 建立资源关系
点击右上角的**【添加关系】**，可在弹窗中建立 CI 间的关联。
- **依赖 (depends_on)**：如 Web 服务依赖数据库。
- **部署在 (runs_on)**：如 Java 服务部署在 ECS 实例上。
- **连接到 (connects_to)**：用于表示网关请求或网络联通。
> [!NOTE]
> 下拉选择框会自动提示资源的分类标签（如：云主机(ECS)），极大降低了连线选错的概率。

## 3. 面向价值的延展 (成本与申请)
*   **资源申请工作流**：支持研发人员直接在平台上发起机器或组件申请表单，并进入审批流。
*   **成本看板 (Cost Report)**：基于资源的关联关系及各自录入的月度成本，平台支持以“业务线”、“环境”、“资源类型”等维度动态生成报表，直观展现云资产的成本去向。
