# SxDevOps 模型供应商配置

## 一、概述

AIOps 智能体的核心依赖是大语言模型（LLM）。平台支持接入多个模型供应商（如 OpenAI、DeepSeek 等兼容 API），为 AI 助手提供推理能力。

**前置条件：**
- 至少一个 LLM API 服务的访问凭证（API Key / Token）
- 需要 `aiops.config.manage` 权限

## 二、平台端配置

### 2.1 进入配置页面

侧边栏：**AIOps → 智能体配置 → 模型供应商** Tab

### 2.2 新增模型供应商

| 字段 | 说明 | 示例 |
|------|------|------|
| 供应商名称 | 自定义名称 | `DeepSeek 生产` |
| 提供商预设 | 预配置的供应商模板 | `DeepSeek` / `OpenAI` / `自定义` |
| API Base URL | API 服务地址 | `https://api.deepseek.com/v1` |
| API Key | 认证密钥（加密存储） | `sk-...` |
| 模型名称 | 使用的具体模型 | `deepseek-chat` / `gpt-4o` |
| 最大 Token 数 | 单次请求的输出上限 | `4096` |
| 输入 Token 价格 | 每百万 Token 价格（用于成本统计） | `$1.00` |
| 输出 Token 价格 | 每百万 Token 价格 | `$2.00` |
| 货币单位 | 价格的货币 | `USD` / `CNY` |
| 启用 / 停用 | 控制供应商是否可用 | 启用 |

### 2.3 支持的供应商

平台预设支持以下供应商模板（自动填写 API Base URL）：

| 提供商 | 默认 API Base URL |
|--------|------------------|
| OpenAI | `https://api.openai.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| 自定义 | 手动填写（兼容所有 OpenAI 兼容 API） |

> 任何兼容 OpenAI Chat Completions API 的服务都可以通过"自定义"模板接入。

### 2.4 多供应商管理

支持配置多个模型供应商。AI 助手工作时：
- 默认使用第一个启用的供应商
- 管理员可在配置中切换默认供应商
- 不同 Skill / Action 可以指定使用的供应商

## 三、成本审计

智能体审计页面自动统计每个供应商的使用数据：
- Token 用量（输入 / 输出）
- 预估费用
- API 调用次数
- 各会话的详细调用记录

费用计算公式：`(输入Token数 × 输入单价 + 输出Token数 × 输出单价) / 1,000,000`

## 四、外部服务要求

### 4.1 兼容性要求

API 服务需要兼容 OpenAI Chat Completions API 格式：
- `POST /v1/chat/completions`
- 支持流式响应（SSE）

### 4.2 推荐供应商

| 供应商 | 说明 |
|--------|------|
| DeepSeek | 国内可直接使用，性价比高 |
| OpenAI | 全球领先，功能丰富 |
| 通义千问 | 阿里云提供，兼容 OpenAI 协议 |
| 本地部署 | 通过 Ollama / vLLM 等部署，填写本地 API 地址 |

## 五、验证

1. 新增模型供应商并保存
2. 打开智能助手页面，发送测试消息："你好"
3. AI 助手正常回复 → 配置成功
4. 查看智能体审计 → 确认调用记录已生成
5. 查看成本统计 → 确认费用计算正确
